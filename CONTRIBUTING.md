# Contributing to Devino

Thank you for contributing.

---

## Before You Start

1. Browse existing Issues.
2. Comment on the issue you want to work on.
3. Wait for assignment before starting.
4. Do not work on issues already assigned.

---

## Branching Strategy

- main → production branch
- develop → integration branch
- feature/<name> → new feature
- fix/<name> → bug fix

Example:

git checkout -b feature/auth-system

---

## Commit Format

Use Conventional Commits:

feat(auth): implement JWT login  
fix(stage): correct unlock logic  
docs: update README  

Keep commits small and meaningful.

---

## Pull Request Process

1. Push your feature branch.
2. Open a PR to develop.
3. Link the related issue.
4. Clearly describe what you implemented.

---

## Quality Standards

- Follow existing code structure.
- Validate inputs properly.
- No debug logs in final code.
- Write clean, readable code.

---

Focus on quality over quantity.
